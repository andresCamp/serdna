# serdna.dev — Changelog

## v2 — Builder Positioning (April 6, 2026)

**Strategic shift:** Freelance-first (10-day window) → bridge to full-time remote product engineer.

### What Changed from v1

**Positioning:**
- From "Senior Product Engineer who ships AI" → "Builder with taste, judgment, and output"
- Primary audience shifts from CTO at established company → founder/lead who needs someone now
- Dual-track: freelance conversion (immediate) + full-time remote (bridge)

**Sections added:**
- **How I Build (Taste Section)** — Four quadrants, full-bleed, 100dvh. Each quadrant: visual + principle + hover reveal. Start with the user, Simplify ruthlessly, Spec-driven development, Judgment over hype.
- **Language Breakdown** — Visual distribution of coding languages across shipped products. TypeScript ~70%, Python ~15%, Swift ~5%, Other ~10%.

**Sections enhanced:**
- **Systems Shipped** — Added "Why this, not that" decision commentary to each technology. Shows architectural judgment, not just usage.
- **Contact** — Headline "Let's build something." Body leads with contract availability + timezone.

**Sections cut (from current implementation):**
- Blog/Signal section (was display-only, no content)
- Values section (replaced by How I Build)
- How I Build video section (replaced by new taste section)
- Talk to Me voice agent (placeholder — removed)
- Internal work routes (`/work/[slug]` — all links go external)

**Unchanged:**
- Hero (minor copy polish only)
- Work bento grid (shipped, polished)
- Open Source cards (shipped)
- Timeline (shipped, copy is solid)
